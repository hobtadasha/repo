Git є стандартом у сучасній розробці, оскільки він забезпечує ефективний контроль версій та спрощує колективну роботу. Він дозволяє відстежувати кожну зміну у коді, швидко повертатися до попередніх стабільних станів та безконфліктно інтегрувати зміни від різних розробників через механізм гілок (branches) та злиття (merging). Це критично важливо для підтримки цілісності та якості програмного забезпечення, особливо у великих та розподілених командах.

Мій досвід після виконання завдання підкреслює зручність постійного створення комітів (commits) з детальними повідомленнями, що перетворює історію проєкту на зрозумілий лог виконаних дій. Найзручнішим виявилось саме створення гілок для ізоляції нових змін, що дозволяє працювати паралельно, не порушуючи роботу основної гілки (main).

У курсовому проєкті Git слід використовувати для збереження історії розробки, що може бути корисним для демонстрації прогресу та відновлення робочого стану перед крайнім терміном. У колективному проєкті Git є незамінним інструментом: кожен член команди повинен працювати у власній гілці для своєї частини функціоналу, регулярно пушити (push) зміни до віддаленого репозиторію (GitHub) та використовувати запити на злиття (Pull/Merge Requests) для рев'ю коду та інтеграції в основну гілку.
